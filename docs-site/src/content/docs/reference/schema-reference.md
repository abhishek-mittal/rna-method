Complete annotated reference for `rna-schema.json` — the canonical source for all RNA Method configurations.

---

## Top-Level Structure

```json
{
  "meta": { ... },
  "agents": [ ... ],
  "rules": [ ... ],
  "skills": [ ... ],
  "hooks": [ ... ],
  "joiningPatterns": [ ... ]
}
```

Validate any `rna-schema.json` against `schema/rna-schema-definition.json` using any JSON Schema validator.

---

## `meta`

Project-level metadata. Required.

```json
{
  "meta": {
    "projectName": "my-app",
    "platform": "copilot",
    "schemaVersion": "1.0.0",
    "lastUpdated": "2026-01-01",
    "director": "Director"
  }
}
```

| Field | Type | Description |
|---|---|---|
| `projectName` | string | Human-readable project name |
| `platform` | enum | `cursor` \| `copilot` \| `claude-code` \| `codex` \| `kimi` |
| `schemaVersion` | string | Semver string (`1.0.0`) |
| `lastUpdated` | string | ISO date string (`YYYY-MM-DD`) |
| `director` | string | Name of the director agent (used in routing tables) |

---

## `agents`

Array of agent definitions. Minimum 1 agent required.

```json
{
  "agents": [
    {
      "id": "developer",
      "name": "Developer",
      "role": "Full-Stack Developer",
      "modelTier": "balanced",
      "file": ".github/agents/developer.agent.md",
      "matchCategories": ["sprint", "blocker"],
      "matchKeywords": ["build", "implement", "fix"],
      "autoApprove": true
    }
  ]
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | ✅ | Unique identifier, used in routing and signal assignment |
| `name` | string | ✅ | Display name for the agent |
| `role` | string | ✅ | Agent's role description (used in routing table) |
| `modelTier` | enum | ✅ | `high-reasoning` \| `balanced` \| `fast` — guides model selection |
| `file` | string | ✅ | Path to the agent's `.agent.md` file (relative to project root) |
| `matchCategories` | string[] | ❌ | Signal categories this agent handles (e.g. `sprint`, `blocker`, `dod`) |
| `matchKeywords` | string[] | ❌ | Keywords that route signals to this agent |
| `contextFiles` | string[] | ❌ | Additional files loaded as context when this agent is active |
| `systemPromptFile` | string | ❌ | Path to a custom system prompt file for this agent |
| `autoApprove` | boolean | ❌ | If `true`, agent can begin work without director sign-off |
| `requiresDirectorApproval` | boolean | ❌ | If `true`, agent must surface a plan before implementing |

### Model Tier Guide

| Tier | Use for | Examples |
|---|---|---|
| `high-reasoning` | Architecture, complex decisions, multi-step planning | architect, director |
| `balanced` | Implementation, review, standard tasks | developer, reviewer |
| `fast` | Research, status reports, routine automation | researcher, ops |

## `rules`

Array of rule definitions. Each rule is an instructions file loaded by the platform.

```json
{
  "rules": [
    {
      "id": "coding-standards",
      "name": "Coding Standards",
      "alwaysApply": true,
      "file": ".github/instructions/coding-standards.instructions.md"
    },
    {
      "id": "security-gate",
      "name": "Security Gate",
      "alwaysApply": false,
      "file": ".github/instructions/security-gate.instructions.md",
      "triggerKeywords": ["security", "auth", "PR", "review"]
    }
  ]
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | ✅ | Unique identifier |
| `name` | string | ✅ | Display name |
| `alwaysApply` | boolean | ✅ | If `true`, rule is loaded for every agent interaction |
| `file` | string | ✅ | Path to the rule's Markdown instruction file |
| `section` | string | ❌ | Section heading within the file (for hub-style files) |
| `triggerKeywords` | string[] | ❌ | Keywords that activate this rule when `alwaysApply` is `false` |

**Tip:** Limit `alwaysApply: true` rules to 2–3 maximum. Each always-apply rule consumes context window on every interaction.

## `skills`

Array of skill definitions. Skills are domain-specific knowledge files loaded on demand.

```json
{
  "skills": [
    {
      "id": "smart-dev-agent",
      "name": "Smart Dev Agent",
      "file": ".github/skills/smart-dev-agent/SKILL.md",
      "canonicalFile": "skills/smart-dev-agent/SKILL.md",
      "ownedBy": "developer",
      "triggerKeywords": ["implement", "build", "fix"]
    }
  ]
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | ✅ | Unique identifier |
| `name` | string | ✅ | Display name |
| `file` | string | ✅ | Platform-specific wrapper path (generated by adapter) |
| `canonicalFile` | string | ✅ | Source of truth for the skill content (validated by `validate-registry.js`) |
| `ownedBy` | string | ✅ | Agent ID that primarily uses this skill |
| `triggerKeywords` | string[] | ❌ | Keywords that load this skill |

## `hooks`

Array of hook definitions. Hooks activate agents or scripts on specific events.

```json
{
  "hooks": [
    {
      "id": "rna-updated-validate",
      "event": "rna-updated",
      "description": "Validate registry when RNA files change",
      "activates": "validate-registry",
      "type": "script",
      "scriptPath": "tools/validate-registry.js",
      "directorApproval": false,
      "enabled": true
    }
  ]
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | ✅ | Unique identifier |
| `event` | string | ✅ | Trigger event (`post-commit`, `pre-push`, `rna-updated`, etc.) |
| `description` | string | ❌ | Human-readable explanation |
| `activates` | string | ❌ | What is activated — a rule ID (for `instruction` type) or a label |
| `type` | enum | ✅ | `script` \| `instruction` |
| `scriptPath` | string | ❌ | Path to the script (required if `type` is `script`) |
| `directorApproval` | boolean | ❌ | If `true`, director must approve before hook fires |
| `enabled` | boolean | ✅ | Master on/off switch. Default new hooks to `false` — enable consciously. |

**Safety default:** Set `enabled: false` for any destructive or side-effect-producing hook until you have verified its behavior.

## `joiningPatterns`

Array of multi-agent pipeline definitions.

```json
{
  "joiningPatterns": [
    {
      "id": "build-review",
      "file": "templates/joins/design-to-build.md",
      "agents": ["developer", "reviewer"],
      "flow": "pipeline"
    }
  ]
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | ✅ | Unique identifier |
| `file` | string | ✅ | Path to the joining pattern documentation file |
| `agents` | string[] | ✅ | Ordered list of agent IDs in the pipeline (minimum 2) |
| `flow` | enum | ✅ | `sequential` \| `pipeline` \| `parallel` |

| Flow Type | Description |
|---|---|
| `sequential` | Agents activate one at a time; each waits for the previous to complete |
| `pipeline` | Each agent receives the output artifacts of the previous agent as input |
| `parallel` | Agents can activate simultaneously (useful for independent workstreams) |

## Versioning Policy

| Change | Version bump |
|---|---|
| Add optional field to existing object | `PATCH` |
| Add new top-level array (backward-compatible) | `MINOR` |
| Remove or rename required field | `MAJOR` |
| Change enum values in existing field | `MAJOR` |

Always update `meta.schemaVersion` and `meta.lastUpdated` when modifying the schema.
