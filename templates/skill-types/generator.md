# Skill Archetype: Generator

> Use this template for skills that **produce artifacts** — files, code,
> documentation, configs, or any other concrete output. Generator skills
> take input parameters, apply a template or logic, and write results.

## Metadata

```json
{
  "id": "{{SKILL_ID}}",
  "name": "{{Skill Name}}",
  "type": "generator",
  "triggerKeywords": ["generate", "create", "scaffold", "write", "produce"],
  "file": "skills/{{skill-id}}.md",
  "skillOutput": "{{output path pattern}}"
}
```

## Template

```markdown
# {{Skill Name}}

**Type:** Generator — produces artifacts.

## Trigger

Activate when the user mentions any of: {{triggerKeywords list}}.

## Inputs

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| target    | yes      | —       | What to generate |
| format    | no       | md      | Output format (md, json, ts, etc.) |
| dest      | no       | auto    | Output directory |

## Protocol

1. **Parse** — Extract parameters from the user request.
2. **Validate** — Confirm required inputs are present. If missing, ask once.
3. **Generate** — Apply the template / logic to produce the artifact.
4. **Write** — Save to the destination path.
5. **Report** — Print confirmation with the file path and a brief preview.

## Template Structure

## Output Rules

- Always create parent directories if they do not exist.
- Never overwrite existing files without explicit confirmation.
- Include a generation header comment in the artifact:
  `<!-- Generated by RNA Method /rna.* — {{SKILL_ID}} — {{ISO date}} -->`

## Boundaries

- Generate only the requested artifact type — do not create side-effect files.
- If the generation depends on another skill's output, declare it in `skillDependencies`.
```

## When to Use

- Component scaffolding
- Documentation generation
- Config file creation
- Test file generation
- Migration scripts
